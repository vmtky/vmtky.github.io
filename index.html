2016年规划

发表于 2017-02-02

岁月不可追，努力在当下。
2015年做了很多计划，想完成，但是没有完成。2016年要继续，记录想完成的事，计划安排，努力去做，到年末总结的时候看是否能够完成。

  1. 工作上：
    • 支持项目的开展，帮忙项目正常交付
    • 完善平台软件，推广使用
    • 熟悉业务知识，形成文档
    • 培训，提高部门人员的业务能力
    • 培养并建设产品部门，完善产品和产品线
  2. 生活上：
    • 照顾好家人，安安，老婆
    • 学车，拿到驾照
    • 办理户口
    • 理财；
  3. 学习上，活到老，学到老：
    • 读书，记录下来；计划每周能够读一本书，写读书笔记。
    • 英语，读写，听力，说全面提高；计划写英文博客，每周一篇文章。
    • 写文章或者博客，每个月有一篇，先这样吧，不知道能不能坚持住。
    • 编程：C#，Html，CSS，Javascript，Python，golang；多写程序，看代码，写技术文章；有人说，教才是最好的学习。
    • 技术：docker，安全；我很佩服黑客，是因为他们技术都很厉害。
    • 管理：沟通，执行力，培训，激励；
    • 反省，特别是自己的思考模式和思考过程。

以上，是想做的事情，时间总会不够用，要想办法，达成目标。

很欣赏一个人，田春冰河（微博）说过：

生活中经常有很多互相冲突的选择，许多事情纷繁复杂、相互影响。一种简单的处理方法是：如果某件事可以肯定是做对了的（例如有多方证据表明从中获得了极大好处），那么所有跟这件事相冲突的其他事——无论看起来多么合理或者受到极大的舆论压力，都应该果断地放弃。#奥卡姆剃刀原理#的一种应用。

此外，计划的执行，需要自律，对于自律，我的理解如下：

  • 没有自律，就没有自由；
  • 按时休息，按时起床，养成规律的生活习惯，形成自己的生物钟；
  • 计划，凡事豫则立；
  • 执行，没有执行力，就是空想，做白日梦，纸上谈兵；
  • 很多事，想，说，容易，做到很难，但也不是不可能，关键看决心，排出困难，顾虑，直接去做就可以了（自己做事老是顾虑太多）；
  • 持之以恒，坚持下去，容易做到不会有成长，克服困难，才会有收获；
  • 找到自己喜欢也擅长做的事，坚持下去；
  • 专注，切记人的精力和时间很有限，长期看没有积累的事情，坚决不做。

以上，就是总体上的目标或者规划，重要的是，要通过努力，理解这个世界，理解自己，找到自己的路。

新的一年，还有一个小小的愿望，一个做起来很简单，实现却很难的愿望，希望能够达成。那就是改变一个人的观念，至少是前进一小步。

2015年年度总结

发表于 2017-02-02

2015年已经过去，站在新年的第二天回顾过去的一年，感觉有点失望，无论是工作还是生活，都没有明显的进步，不知道未来的路在何方，只能随波逐流，看风把我吹向何方。以下是去年的计划和总结。

2015年年初计划

14年做了很多计划，想完成，但是没有完成。2015年要继续，记录想完成的事，计划安排，年末看是否能够完成。

  1. 工作上：
    • 完成Framework的设计，开发其他项目上，按项目计划做；
    • 没有下来的项目，无法提前做计划；
  2. 生活上：
    • 买房子；
    • 安排安安宝贝上幼儿园；
    • 学车，拿到驾照；
    • 下半年软考（如果户口能够落下来，则不必花费时间和精力再考试）；
    • 理财；
    • 办护照，台湾入台证和通行证；
  3. 学习上，活到老，学到老：
    • 读书，记录下来；
    • 英语，读写，听力，说全面提高；
    • 编程：C#，Html，CSS，Javascript，Python，golang；
    • 技术：docker，安全；
    • 管理：沟通，执行力，培训，激励；
      以上，只是想做的事情，具体的、能够执行的计划，稍后排出。

以上的很多事，还是没有完成。
工作上的事情，基本完成；生活上的事情，学车没有开始，软考后来发现中级的证书已经通过，没有考高级的，理财基本没有开始；学习上，有读书，不连续，其他基本没有进步；给人虚度光阴的感觉。
很平凡的一年，有不如意的事情，比如弟弟的工作不如意，小孩夭折；有小小的成绩，比如申请专利，买房子；现在想来，都不知道这一年，做了什么事情，恍然若梦，虚度光阴。但是年末的时候，看了李笑来和TinyVoice的文章，觉得很多事还是可以做的，什么时候都不晚，时间还很长，慢慢去成长，找到自己的节奏，学习，生活。

补记：这是2016年1月2日写的2015年年终总结，中间隔了一年的时间都没有再写任何的技术博客文章，希望这次不要这样。

Hello GitHub

发表于 2017-02-02

经过几天简单的尝试，博客可以显示简单的内容了，而格式还需要再改进。
很久没有接触web相关的编程，都忘记了，以后再慢慢熟悉吧。
在这个地方，我想记录自己学习的内容、阅读的书籍、一些想法和总结，以写作促进思考。
只有思维的成长，才是重要的成长，否则只是年龄变大，身体变老而已。

这篇文章实际的发布时间是2014/12/30写的，因为技术转换，把之前发布的文章全部删除，从新发布一下。
2014年底，开始接触GitHub Pages，才开始想写技术类的文章帮忙自己成长，可惜没有坚持下来。那时使用的技术是Jekyll，基于Ruby开发的。两年时间过去了，技术发展一日千里，日新月异，JS和前端如日中天。很长时间以来都想了解前端的各种技术，但是种种原因，都没有付诸实施。现在公司的部分产品也要转成BS架构，也要支持手机和平板，因此了解前端技术势在必行。2017年春节假期没有回家过年，空余时间了解前端、NodeJS、HTML、CSS、JS等各种知识，也想从新捡起GitHub Pages的博客写作，故把技术从Jekyll转换成基于NodeJS的Hexo，也算是一种实践吧，此为补记。
